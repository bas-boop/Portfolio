Description

Dit is een groot project waarin ik een developer ben. Het is een onderdeel van het HLO (hybride leer omgeving).
Dit houdt in dat voornamelijk studeten er aan werken, naast 2 docenten/product owners.
Elke 10 weken wordt er een nieuwe selectie aan studenten gekozen, je kan meerdere keren gekozen worden.
De game is een 2.5D Metroidvania Couch Co-op gebaseerd op de cartoons van 80's Sci-Fi. Het is een game dat gaat om momentum.
Wat interessant is om te vertellen, is dat deze game zijn eigen force systeem heeft.
Dat vergelijkbaar is met Rigidbody wat in Unity zit, maar dan beter naar mijn mening.

Wat ik ook heb gedaan is voor 1 sprint scrum master zijn, voor 2 weken.
Dus de daylie stand-ups en -downs organiseren en door de dag heen bij iedereen checken of het lukt met wat ze maken.
Ook kijk ik of iedereen wel de todo’s op Trello goed hebben staan.

Wat ik vooral geleerd heb is in een groot team werken en een betere programmeur worden.
Want we verdiepen in game development wat ik niet zou bereiken met een simpel 8 weken school project.

Heb aan het zelfde project ook gewerkt toen werkte ik vooral aan Custom Physics System

De git repository is op privé!
Het wordt een commerciële game.

Jump momentum (Version 1):

Before:

After:

Wanneer je tijdens het lopen zou springen en dan lopen los laat dan zal je naar benden vallen, alsof je een onzichtbare muur zal raken.
Beetje onlogische, ik heb er voor gezorgd dat je momentum zal blijven houden.
Dit heb ik gedaan met een force te geven wanneer je geen input meer geef.


Jump momentum (Version 2):

Bij de eerste versie verloor je wat controlle,
want je krijgt een zet distance mee geven, die je wel kan om draaien (meerdere keren).
Alleen je kon geen tap geven en een beetje verder komen.

Daarom verander ik de air accleration dynamis met wel of niet input geven in de lucht.

Release ledge:

Optimaliseren van het loslaten van de ledge wanneer je er aan hangt.
Je kon er namelijk alleen boven op komen, dus we hebben het gemaakt dat je het gewoon kan loslaten, waardoor je naar beneden valt.
Dit geeft meer opties en logica aan het spel.



Decelerator:

Als je van een hoogte zal komen naar de grond, dan zou je momentum houden wanneer je land op de grond.
Hierdoor sta je niet ineens stil op de grond wanneer je heel snel ging.
Dit hebben we gedaan via de forces van de Force System.


Deceleration:

Heb ook gewerkt aan de Deceleration van de speler.
Dus als je aan het lopen bent en stopt met input geven, dan doe je nog een paar kleine stappen totdat je echt stil staat.
Hierdoor sta je niet gelijk stil alsof je een muur raakt.
