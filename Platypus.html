Tatum heeft veel geholpen met het design.

Abstract, player and enemy StateMachine:

Ik had niet genoeg controle over de player vond ik, dus ik heb een StateMachine ervoor gemaakt om meer controle over te hebben.
Als ik toch bezig ben waarom niet een abstract StateMachine en gelijk enemy StateMachine maken.
Er is ook een state queue gemaakt, dit is vooral handig voor de speler. Was je net te vroeg met je input, dan wordt het alsnog gedaan.
Dit heeft een grote focus op S.R.P., anders kon ik de hele speler in 1 class maken.
Wat ik hiervan heb geleerd een betere begrip over abstract classes en diepe duik in code nemen vind ik heel leuk.

UML class diagram PDF!

Pick-up System:

Dit is een pick-up system met een PickType als Enum. Ook heb je een inleverpunt om de opgepakt items in te leveren.
Dit heeft een goede implantatie van SRP.

Player Dash:

De feature waardoor het project is gestart.
Dit lijkt heel erg op een debug feature die ik had gemaakt tijdens Custom Physics System, het leek me leuk om het terug te brengen.

Wanneer je klikt op het scherm daar dashed de speler naartoe.
